Add evidence-grounded resume content planning.

Context:
The Professional Knowledge Engine currently supports:
- canonical professional knowledge
- reconciliation and trust policies
- hybrid retrieval
- job analysis
- Candidate Evidence Packs
- LLM-based Evidence Reasoning
- Curated Evidence Packs
- evaluation and observability

Goal:
Generate a structured Resume Content Plan exclusively from a Curated Evidence Pack.

Architecture:
Curated Evidence Pack
→ Resume Content Planner
→ Resume Content Plan
→ Renderer

Scope:
- Implement or complete the documents module using hexagonal boundaries
- Add ResumeContentPlanner port
- Add LlmResumeContentPlanner implementation
- Reuse the existing LlmProvider
- Add ResumeContentPlan
- Add PlannedSummary
- Add PlannedExperience
- Add PlannedBullet
- Add PlannedSkillGroup
- Add OmittedEvidence
- Add CLI command:
  - pke documents resume plan <job-id>
- Add options:
  - --model
  - --language pt-BR|en
  - --length concise|standard|detailed
  - --json
  - --verbose
- Use the latest compatible Curated Evidence Pack for the job
- Require structured output validated through Zod or equivalent
- Persist immutable resume plans
- Version plans by:
  - curatedEvidencePackId or hash
  - provider
  - model
  - promptVersion
  - language
  - length
- Add prompt versioning
- Add OpenTelemetry and Langfuse instrumentation
- Add deterministic validation after generation
- Add golden evaluation scenarios
- Update architecture, documents and roadmap documentation

ResumeContentPlan must include:
- jobDescriptionId
- curatedEvidencePackId
- language
- length
- professional summary
- planned experiences
- planned bullets
- planned skill groups
- selected evidence IDs
- omitted evidence IDs and reasons
- uncovered requirements
- warnings
- provider
- model
- promptVersion
- createdAt

PlannedBullet must include:
- text
- supportingEvidenceIds
- targetRequirementIds
- source organization or experience ID
- exaggerationRisk
- warnings

Rules:
- The planner receives only the Curated Evidence Pack and canonical presentation metadata needed to render selected evidence
- It must not access repositories, pgvector, external search or unrestricted tools
- Every factual sentence must reference at least one selected canonical EvidenceClaim
- Metrics must be preserved exactly
- Dates, companies, roles and technologies must not be changed
- Missing requirements must remain missing
- Weak evidence must not be rewritten as strong ownership
- Skill-only evidence must not be converted into production experience
- The planner may omit evidence for relevance or length
- The planner may rewrite wording but not factual meaning
- Invalid or unsupported bullets must fail deterministic validation
- Content generation and visual rendering remain separate
- Prefer omission over unsupported claims

Deterministic validators:
- every bullet has supporting evidence
- every supporting evidence ID exists in the Curated Evidence Pack
- no rejected or discarded evidence is used
- metrics in generated text match source metrics
- company and role names match canonical values
- no unsupported technology is introduced
- no uncovered requirement is represented as covered
- output respects requested language and approximate length

Initial output:
- JSON Resume Content Plan only
- Optional compact terminal preview

Out of scope:
- PDF rendering
- DOCX rendering
- visual templates
- cover-letter generation
- LinkedIn generation
- interview-answer generation
- automated application submission
- LLM benchmarking

Acceptance criteria:
- pke documents resume plan <job-id> produces a validated Resume Content Plan
- every factual bullet is traceable to selected evidence
- unsupported facts are rejected
- concise and detailed modes produce different bounded sizes
- Portuguese and English plans are supported
- repeated identical requests reuse an immutable existing plan
- evaluation detects fabricated evidence and altered metrics
- existing workflows remain functional
