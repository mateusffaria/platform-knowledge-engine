Evolve the Canonical Career Model to support richer professional knowledge.

Context

The current canonical model was sufficient for validating ingestion and semantic retrieval, but it no longer represents the richness of real professional profiles.

Real resume sources contain organizations, roles, products, initiatives, metrics, education, certifications, responsibilities and relationships between entities.

The project must evolve towards a canonical domain model instead of mirroring resume sections.

Goal

Separate persistent professional entities from atomic evidence claims.

The canonical model must represent professional knowledge independently of how resumes are structured.

Scope

Replace the current simplified taxonomy with:

KnowledgeAsset

- ProfessionalProfile
- Organization
- ProfessionalExperience
- Role
- Project
- Initiative
- Product
- Education
- Certification
- Skill

EvidenceClaim

Represent atomic assertions about KnowledgeAssets.

Introduce explicit claim categories:

- fact
- responsibility
- achievement
- metric
- capability
- relationship

Claims must reference their subject instead of embedding complete objects.

Example

ProfessionalExperience
↓

Achievement

↓

Metric

↓

Technology relationship

instead of

Experience
↓

Huge text blob

Add structured predicates such as:

works_at
holds_role
uses_technology
participated_in
occurred_during
reduced_processing_time
reduced_cost
improved_reliability
demonstrates

Preserve

- provenance
- original excerpt
- source language
- original section label

Document sections must not become canonical entity types.

The canonical model represents professional knowledge, not resume formatting.

Migration

Maintain backwards compatibility where possible.

Acceptance Criteria

- richer canonical entities
- atomic claims
- multilingual sources
- existing ingestion still works
- reconciliation continues functioning
- semantic search remains compatible
