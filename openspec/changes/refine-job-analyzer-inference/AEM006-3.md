Refine JobAnalyzer inference boundaries and signal taxonomy.

Context:
The first JobAnalyzerAgent is functional and returns schema-validated analysis with source provenance, warnings, provider metadata and prompt versioning.

Observed issues:
- A narrow ambiguity requirement was expanded into unsupported communication and stakeholder-management competencies.
- Cross-team collaboration and cross-team leadership are currently grouped together.
- Domain signals may contain duplicate or non-canonical variations.
- Seniority signals lose useful source context.

Goal:
Make agent interpretation more conservative, normalized and suitable for downstream retrieval.

Scope:
- Update the JobAnalyzer prompt to require the narrowest defensible inference
- Explicitly prohibit unsupported competency expansion
- Separate:
  - crossTeamCollaborationSignals
  - crossTeamLeadershipSignals
- Add canonical and source values to domain signals
- Normalize equivalent domain variations
- Refine seniority signals to include:
  - canonical level
  - source value
  - signal type
  - source reference
- Preserve warnings for weak or ambiguous inferences
- Add tests for conservative inference
- Add tests distinguishing collaboration from leadership
- Add tests for domain normalization
- Add tests for jobs without explicit seniority
- Define idempotency/versioning behavior for repeated analysis
- Update the Job Analyzer documentation and prompt version

Rules:
- The agent must prefer omission over unsupported inference
- Source references do not make an inference valid by themselves
- Warnings must not be used to excuse clearly unsupported output
- Canonical normalization must preserve original source wording
- Existing persisted analyses must remain readable

Acceptance criteria:
- Ambiguity does not imply stakeholder management without supporting evidence
- Collaboration and leadership signals are distinct
- Equivalent domain terms are normalized
- Seniority retains source context
- Repeated analysis follows a documented deterministic versioning policy
- Existing tests continue passing
