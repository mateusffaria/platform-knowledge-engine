Introduce PKQL (Professional Knowledge Query Language).

Context

The retrieval module currently relies on heuristic query planning.

As the project evolves, both humans and agents should be able to express deterministic retrieval intents.

Instead of interpreting arbitrary text everywhere, the system should parse queries into a canonical representation before planning retrieval.

Goal

Introduce PKQL as the canonical query language of the Professional Knowledge Engine.

PKQL must support both human-written queries and agent-generated queries.
