Add the first agentic workflow for job analysis.

Context:
The project currently supports:
- professional knowledge ingestion
- trusted knowledge reconciliation
- semantic indexing
- PKQL
- metadata-driven hybrid retrieval
- canonical job description ingestion
- deterministic requirement extraction
- Evidence Pack generation

Goal:
Introduce the first specialized agent, JobAnalyzerAgent, to enrich the canonical job model while preserving deterministic extraction, provenance and architectural boundaries.

Architecture:
Job Source
→ Deterministic Job Parser
→ Canonical Job Model
→ JobAnalyzerAgent
→ Enriched Job Analysis
→ Retrieval Intent
→ Hybrid Retrieval
→ Evidence Pack

Scope:
- Add an LlmProvider application port
- Add an OllamaLlmProvider infrastructure adapter
- Add LlmProviderFactory at the composition root
- Configure:
  - LLM_PROVIDER=ollama
  - LLM_MODEL
  - OLLAMA_BASE_URL
- Add JobAnalyzer port or application service
- Add JobAnalyzerAgent implementation
- Add JobAnalysis domain/application model
- Allow the agent to identify:
  - inferred requirements
  - seniority signals
  - domain signals
  - cross-team leadership signals
  - architecture and reliability expectations
  - ambiguities and warnings
- Preserve deterministic requirements separately from inferred requirements
- Require structured output validated with Zod or equivalent
- Add CLI command:
  - pke jobs analyze <job-id>
- Add optional:
  - --model
  - --json
  - --verbose
- Persist agent analysis separately from the original JobDescription
- Add Langfuse tracing through the existing observability port
- Add tests using mocked LlmProvider responses
- Update architecture, jobs and agent documentation
- Add ADR for the first agentic workflow

Rules:
- Deterministic extraction remains authoritative for explicit source text
- Agent-inferred requirements must be marked as inferred
- Agent outputs must preserve references to job source excerpts where possible
- The agent must not modify professional EvidenceClaims
- The agent must not access repositories, PostgreSQL, pgvector or Ollama directly
- The agent must not resolve truth conflicts
- Structured output must be schema-validated
- Invalid agent output must fail safely and preserve the original job model
- LLM calls must be observable
- Agent prompts must be versioned

Out of scope:
- resume generation
- cover letter generation
- multiple-agent orchestration
- autonomous tool loops
- OpenAI provider
- LLM benchmarking
- LLM-based professional claim validation
- automatic job application

Acceptance criteria:
- pke jobs analyze <job-id> produces a validated JobAnalysis
- explicit and inferred requirements remain distinguishable
- invalid LLM output is rejected safely
- analysis can be used to build a retrieval intent
- retrieval still returns a traceable Evidence Pack
- Ollama remains behind the LlmProvider port
- existing deterministic behavior remains functional
- existing tests continue passing
