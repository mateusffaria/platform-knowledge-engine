Canonical professional profile input:

The MVP must define `professional-profile/v1` as the canonical
candidate input schema.

The schema must be represented as Markdown with YAML front matter and
explicitly labeled fields.

Required front matter:

- schema: professional-profile/v1
- language: en or pt-BR

Required candidate field:

- Name

Optional candidate fields:

- Headline
- Location
- Email
- Phone
- LinkedIn
- GitHub
- Website

Canonical knowledge sections:

- Candidate
- Professional Summary
- Professional Experience
- Technical Skills
- Education
- Certifications

The ingestion pipeline must parse Candidate metadata deterministically
and persist it using the existing knowledge asset metadata mechanism.

Candidate presentation metadata must not be modeled as evidence claims.

Resume generation must load presentation metadata from the canonical
profile and content sections from the existing ResumeContentPlan.

Missing optional contact fields must not prevent resume generation.

A missing candidate name and a missing renderable experience must
produce separate actionable validation errors.

PDF and DOCX profile ingestion remain out of scope for the MVP.
Existing source resumes must be manually normalized into the canonical
Markdown format.
