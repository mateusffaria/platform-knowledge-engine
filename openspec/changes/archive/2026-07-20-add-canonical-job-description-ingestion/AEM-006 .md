Add canonical job description ingestion and requirement extraction.

Context:
The project currently supports:
- professional knowledge ingestion
- trusted knowledge reconciliation
- semantic indexing
- PKQL
- metadata-driven query planning
- hybrid retrieval
- Evidence Pack generation

Goal:
Introduce a jobs module that transforms job descriptions into a canonical Job Model and produces retrieval intents for the Professional Knowledge Engine.

Architecture:
Job Source
→ Job Parser
→ Canonical Job Model
→ Job Requirements
→ Retrieval Intent
→ Hybrid Retrieval
→ Evidence Pack

Scope:
- Create src/modules/jobs using hexagonal boundaries
- Add JobDescription domain entity
- Add JobRequirement domain entity
- Add JobRequirementType:
  - skill
  - technology
  - experience
  - responsibility
  - seniority
  - domain
  - education
  - language
- Add requirement importance:
  - required
  - preferred
  - inferred
- Add JobSourceParser port
- Add deterministic Markdown/plain-text job parser
- Preserve source excerpts and source locations
- Add IngestJobDescription use case
- Add BuildJobRetrievalIntent use case
- Produce PKQL filters and semantic text from normalized requirements
- Add CLI commands:
  - pke jobs ingest <file>
  - pke jobs show <job-id>
  - pke jobs retrieve <job-id>
- Reuse the existing retrieval module through application contracts
- Persist job descriptions and requirements
- Add migrations and tests
- Update architecture and roadmap documentation

Rules:
- Jobs module owns job descriptions and job requirements
- Retrieval module owns retrieval execution and Evidence Pack generation
- Jobs module must not import retrieval infrastructure
- Requirement extraction must preserve provenance
- Inferred requirements must be explicitly marked as inferred
- Deterministic extraction must not fabricate missing requirements
- PKQL generation must use canonical query fields
- Unknown requirement values remain semantic text rather than being discarded

Initial extraction behavior:
- Detect explicit sections such as:
  - Requirements
  - Qualifications
  - Responsibilities
  - Nice to have
  - Preferred qualifications
- Classify bullet points by section context
- Match persisted professional metadata where useful
- Preserve unmatched requirement text for semantic retrieval

Out of scope:
- LLM-based job analysis
- URL scraping
- ATS scoring
- resume generation
- cover letter generation
- automatic job application
- benchmarking

Acceptance criteria:
- A Markdown or text job description can be ingested
- Requirements are persisted with source excerpts
- Required and preferred requirements are distinguished
- A retrieval intent can be generated
- pke jobs retrieve <job-id> returns an Evidence Pack
- Existing retrieval behavior remains unchanged
- Existing tests continue passing
