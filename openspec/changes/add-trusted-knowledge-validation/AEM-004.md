Add trusted knowledge validation and conflict resolution.

Context:
The system can ingest professional knowledge, persist it, generate embeddings and perform semantic search. It currently lacks an explicit trust model for claims coming from multiple sources.

Goal:
Introduce claim validation, conflict detection and a human-review workflow so that only eligible professional knowledge is indexed and retrieved as trusted evidence.

Scope:
- Extend EvidenceClaim with:
  - status
  - confidenceScore
  - conflictSeverity
  - reviewedAt
  - reviewReason
- Add claim statuses:
  - confirmed
  - single_source
  - needs_review
  - rejected
  - superseded
- Add ConflictDetector port or domain service
- Add deterministic conflict detection rules
- Distinguish missing evidence from contradictory evidence
- Add source priority or source reliability metadata
- Add use case to assess newly ingested claims
- Add use case to list claims requiring review
- Add use case to confirm or reject a claim
- Add CLI commands:
  - pke claims review
  - pke claims confirm <claim-id>
  - pke claims reject <claim-id> --reason "<reason>"
- Prevent rejected and superseded claims from semantic indexing
- Decide and document whether single_source claims are searchable by default
- Reindex or remove embeddings when claim eligibility changes
- Add migrations and tests
- Update architecture and roadmap documentation

Domain rules:
- Absence of a claim in another source is not a conflict
- Different values for the same factual attribute may be a conflict
- User confirmation overrides automated confidence scoring
- Rejected claims must never be used in generated outputs
- Superseded claims remain auditable but are not active
- All status changes must preserve traceability

Out of scope:
- LLM-based conflict resolution
- Web interface
- Resume generation
- Full hybrid retrieval
- Automatic merging of high-severity conflicts
- Benchmarking

Acceptance criteria:
- Conflicting claims are marked as needs_review
- Missing information is not incorrectly marked as conflict
- Claims can be confirmed or rejected from the CLI
- Rejected and superseded claims are excluded from indexing
- Existing valid embeddings remain searchable
- Status changes are persisted and auditable
- Tests cover no-conflict, low-conflict and contradictory cases

Documentation:
- Move detailed trusted-knowledge behavior out of README
- Create docs/trusted-knowledge.md
- Create docs/adr/0002-trusted-knowledge-policy.md
- Keep README concise and add links to detailed documentation
- Document clearly that absence of evidence is not contradiction
- Document the default search and generation eligibility policy by claim status
- Document that deterministic rules and user review take precedence over LLM judgment
