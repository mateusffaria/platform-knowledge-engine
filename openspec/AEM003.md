Add semantic indexing and vector search using pgvector.

Goal:
Create the retrieval foundation by generating embeddings from verified professional knowledge and storing them in PostgreSQL using pgvector.

Scope:
- Create knowledge_embeddings table
- Add EmbeddingProvider port
- Add VectorStore port
- Add deterministic text builder for KnowledgeAssets/EvidenceClaims
- Add pgvector adapter
- Add CLI command: pke index
- Add CLI command: pke search "<query>"
- Add tests for embedding text generation
- Add tests for vector search flow using mocked embeddings

Rules:
- Embedding text must be deterministic
- Vector search must not become the source of truth
- Embeddings must reference the original KnowledgeAsset or EvidenceClaim
- Re-running pke index must be idempotent
- Do not index rejected or unverified claims as confirmed facts

Out of scope:
- Resume generation
- Cover letter generation
- Agent orchestration
- Benchmarking
- Multiple provider comparison
- Full hybrid retrieval ranking
- UI/dashboard

Acceptance criteria:
- Persisted EvidenceClaims or KnowledgeAssets can be indexed
- Running pke index twice does not duplicate embeddings
- pke search "<query>" returns related evidence
- Existing ingestion behavior remains unchanged
- Existing tests continue passing
